## typescript patterns to document via briefs

### genBrowserAuthSession [DOCUMENT]

**location**: `src/logic/browser/genBrowserAuthSession.ts`

**pattern**: mutable browser session handle with respawn capability for mode switch

**key interface**:
```typescript
interface BrowserAuthSession {
  browser: Browser;
  context: BrowserContext;
  mode: BrowserMode; // 'HEADLESS' | 'HEADFUL'
  close: () => Promise<void>;
  respawn: (input: { mode: BrowserMode }) => Promise<void>;
  storage: {
    get: () => Promise<void>;  // persist to disk
    set: () => Promise<void>;  // restore from disk
  };
}
```

**why important**:
- enables captcha handling in headful → handoff to headless automation
- preserves authentication across mode switch via storageState
- mutable handle pattern for long-lived sessions

**verdict**: document via brief with code examples, reference declastruct-squarespace

### withRemoteStateCache [DOCUMENT]

**location**: `src/logic/squarespace/withRemoteStateCache.ts`

**pattern**: cache context for browser scrape operations with automatic invalidation

**key implementation**:
```typescript
const remoteStateCacheContext = createRemoteStateCacheContext<
  [any, ContextSquarespaceAgent],
  SimpleOnDiskCache
>({
  cache: ({ fromInput }) => fromInput[1].agentOptions.remoteState.cache,
  serialize: {
    key: (input, context) => [...].join('.'),
    value: (output) => serialize(output, { lossless: true }),
  },
});

// usage:
export const getProductBySku = withRemoteStateQueryCache(remoteStateCacheContext)(
  async (input: { sku: string }, context: ContextSquarespaceAgent) => {
    // expensive browser scrape here
  }
);
```

**npm package**: `with-remote-state-cache`

**why important**:
- browser scrapes are expensive (network, render time, rate limits)
- queries wrap with cache, mutations trigger invalidation
- prevents redundant page loads for repeated queries

**verdict**: document via brief with usage examples
