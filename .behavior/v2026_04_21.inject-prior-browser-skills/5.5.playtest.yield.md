# playtest: browser automation skills and briefs

## prerequisites

- node.js 18+ installed
- pnpm installed
- repo cloned and dependencies installed (`pnpm install`)
- playwright chromium installed (`npx playwright install chromium`)
- display available (for headful browser tests)

## sandbox

all file operations target `.temp/` directory:
- `.temp/playtest/` — playtest artifacts
- `.cache/browser.playtest/` — browser session cache (auto-created by skills)

## happy paths

### 1. browser lifecycle: start → describe → stop

**action:**
```bash
# start a browser session
rhx browser.start --session playtest --mode HEADLESS

# describe open tabs
rhx browser.describe --session playtest

# stop the browser
rhx browser.stop --session playtest
```

**expected outcome:**
- `browser.start` prints session started message with ws-endpoint
- `.cache/browser.playtest/ws-endpoint` file is created
- `browser.describe` lists tabs (at least 1 tab at about:blank or similar)
- `browser.stop` prints session stopped message
- `.cache/browser.playtest/ws-endpoint` file is removed

**acceptance test:** `browser.lifecycle.integration.test.ts`

---

### 2. browser action: run playbook

**action:**
```bash
# ensure browser is active
rhx browser.start --session playtest --mode HEADLESS

# create a test playbook
mkdir -p .temp/playtest
cat > .temp/playtest/goto-example.play.ts << 'EOF'
import type { Page, Browser } from 'playwright';

export const action = async (input: { page: Page; browser: Browser }) => {
  await input.page.goto('https://example.com');
  return { title: await input.page.title() };
};
EOF

# run the playbook
rhx browser.action --session playtest --play .temp/playtest/goto-example.play.ts

# stop browser
rhx browser.stop --session playtest
```

**expected outcome:**
- `browser.action` executes playbook without error
- output shows `{ title: 'Example Domain' }` or similar

**acceptance test:** `browser.action.integration.test.ts`

### 6. scraper cache: expire and extend

**action:**
```bash
# create a test cache entry
mkdir -p .cache/scraper.playtest
echo '{"data":"test"}' > .cache/scraper.playtest/test-entry.json

# expire a cache entry
rhx cache.expire --session playtest --key test-entry

# verify entry is removed
ls .cache/scraper.playtest/test-entry.json 2>&1 || echo "file removed as expected"

# extend cache (keep entry alive - create new entry first)
echo '{"data":"test2"}' > .cache/scraper.playtest/test-entry2.json
rhx cache.extend --session playtest --key test-entry2 --ttl 3600

# cleanup
rm -rf .cache/scraper.playtest
```

**expected outcome:**
- `cache.expire` removes the cache entry
- `cache.extend` extends TTL without error

**acceptance tests:**
- `cache.expire.integration.test.ts`
- `cache.extend.integration.test.ts`

### 8. full user journey

**action:**
```bash
# complete journey: start → navigate → snapshot → diagnose → stop
rhx browser.start --session playtest --mode HEADLESS

# navigate to a page
cat > .temp/playtest/navigate-and-verify.play.ts << 'EOF'
import type { Page, Browser } from 'playwright';

export const action = async (input: { page: Page; browser: Browser }) => {
  await input.page.goto('https://example.com');
  const h1text = await input.page.locator('h1').textContent();
  return { h1text };
};
EOF

rhx browser.action --session playtest --play .temp/playtest/navigate-and-verify.play.ts

# take diagnostic snapshot
rhx browser.snapshot --session playtest --focused

# grep HTML for expected content
grep -o 'Example Domain' .cache/browser.playtest/snapshot.*/snapshot.html | head -1

# cleanup
rhx browser.stop --session playtest
rm -rf .temp/playtest
```

**expected outcome:**
- entire journey completes without error
- playbook returns h1text "Example Domain"
- snapshot captures page state
- grep finds expected content in HTML

**acceptance test:** `browser.journey.integration.test.ts`

## edgey paths

### 1. no browser active

**action:**
```bash
# try to describe without start browser first
rhx browser.describe --session nonexistent
```

**expected behavior:**
- command fails fast with clear error message
- error indicates no browser active for session

---

### 2. invalid playbook path

**action:**
```bash
rhx browser.start --session playtest --mode HEADLESS
rhx browser.action --session playtest --play .temp/nonexistent.play.ts
rhx browser.stop --session playtest
```

**expected behavior:**
- `browser.action` fails with clear error about absent file
- browser session remains active (action failure doesn't crash browser)

---

### 3. snapshot without --focused or --tab

**action:**
```bash
rhx browser.start --session playtest --mode HEADLESS
rhx browser.snapshot --session playtest  # no --focused or --tab
rhx browser.stop --session playtest
```

**expected behavior:**
- command fails with usage error that requires --focused OR --tab
- helps prevent wrong-tab snapshot mistakes

---

### 4. double start (idempotent)

**action:**
```bash
rhx browser.start --session playtest --mode HEADLESS
rhx browser.start --session playtest --mode HEADLESS  # second start
rhx browser.stop --session playtest
```

**expected behavior:**
- second start either succeeds (reconnects) or fails clearly
- no orphan browser processes

---

### 5. stop without start

**action:**
```bash
rhx browser.stop --session nonexistent
```

**expected behavior:**
- command fails gracefully or succeeds as no-op
- no crash or hang

---

### 6. cache expire on absent key

**action:**
```bash
# try to expire a key that doesn't exist
rhx cache.expire --session playtest --key nonexistent-key
```

**expected behavior:**
- command succeeds as no-op (idempotent)
- no error thrown for absent key

---

### 7. cache extend on absent key

**action:**
```bash
# try to extend a key that doesn't exist
rhx cache.extend --session playtest --key nonexistent-key --ttl 3600
```

**expected behavior:**
- command fails with clear error about absent key
- OR succeeds as no-op (depends on implementation)

---

## pass/fail criteria

### pass if:
- [ ] all 8 happy path sections complete without error
- [ ] all 7 edgey path sections behave as expected
- [ ] snapshot artifacts (PNG, HTML, JSON) are valid and readable
- [ ] briefs are coherent and teach the documented patterns
- [ ] no orphan browser processes after all tests
- [ ] no leftover artifacts outside .temp/ and .cache/

### fail if:
- [ ] any happy path step fails unexpectedly
- [ ] browser crashes on playbook execution
- [ ] snapshot files are empty or corrupted
- [ ] briefs are absent or incomplete
- [ ] orphan browser processes remain after stop
- [ ] tests pollute directories outside sandbox
