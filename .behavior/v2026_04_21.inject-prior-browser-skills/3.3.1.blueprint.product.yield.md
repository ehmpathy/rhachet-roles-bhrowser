## implementation notes

### skill dispatcher pattern

follow rhachet-roles-ehmpathy dispatcher pattern for browser.snapshot subcommands:

```bash
# browser.snapshot.sh dispatches to subcommands
case "$1" in
  screen|html|console|storage|network|meta)
    exec "$SCRIPT_DIR/browser.snapshot.$1.sh" "${@:2}"
    ;;
  *)
    # capture all assets
    ;;
esac
```

### test infrastructure

clone `.test/infra/browser.ts` with:
- `CACHE_ROOT = '.temp/.cache'` for isolation
- `rhxFull(cmd)` for skill execution
- `stopBrowser(session)` for cleanup
- `asStableSnapshot(output)` for deterministic snapshots

### exit code semantics

per rule.require.exit-code-semantics:

| exit code | category | when |
|-----------|----------|------|
| 0 | success | operation completed |
| 1 | malfunction | external error (network, permission, unexpected state) |
| 2 | constraint | user must fix (invalid args, session not found, tab not found) |

### error output format

per rule.require.failloud — errors must include actionable hints:

| error | hint |
|-------|------|
| session not found | "session 'X' not found. run browser.start --session X first" |
| tab not found | "tab N not found. run browser.describe --session X to list tabs" |
| playbook not found | "playbook 'X' not found at path Y" |
| url mismatch | "expected url 'X' but found 'Y'. verify you selected the correct tab" |
| browser not active | "no browser found for session X. was it stopped or crashed?" |
| storage state not found | "no session state found for 'X'. run browser.session set --from @storage first" |
| source file not found | "storage file 'X' not found at path Y" |
