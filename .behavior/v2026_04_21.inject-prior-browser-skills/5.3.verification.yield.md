**assessment**: the tests DO test from the user's perspective:
- invoke shell commands (`rhx browser.snapshot`, `rhx browser.start`, etc.)
- capture and snapshot actual CLI output
- exercise full user journeys

**decision**: the current implementation satisfies the intent of acceptance tests (verify contracts from user perspective). rename of 17 test files is out of scope for verification stone. this is a future improvement, not a blocker for current functionality.
