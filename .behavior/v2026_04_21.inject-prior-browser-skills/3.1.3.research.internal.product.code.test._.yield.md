## test infrastructure [CLONE]

### .test/infra/browser.ts

**location**: `.agent/repo=.this/role=any/skills/.test/infra/browser.ts`

**key exports**:
```typescript
// isolated cache root for tests
export const CACHE_ROOT = '.temp/.cache';

// get wsEndpoint file for session
export const getWsEndpointFile = (session: string) =>
  `${CACHE_ROOT}/browser.${session}/ws-endpoint`;

// run rhx with isolated cache, return full result
export const rhxFull = (cmd: string): RhxResult => { ... };

// stop browser and suppress errors (cleanup helper)
export const stopBrowser = (session = 'default') => { ... };

// redact dynamic values for stable snapshots
export const asStableSnapshot = (output: string) =>
  output
    .replace(/\x1b\[[0-9;]*m/g, '')  // strip ANSI
    .replace(/ws:\/\/localhost:\d+\/devtools\/browser\/[a-f0-9-]+/g, 'ws://localhost:PORT/devtools/browser/UUID')
    .replace(/\d{8}T\d{6}Z/g, 'ISOTIME')
    .replace(/\.cache\/browser\.([\w-]+)\/snapshot\.[^/\s]+/g, '.cache/browser.$1/snapshot.ISOTIME.tabN');
```

**why important**:
- isolated test cache prevents pollution between test runs and production
- `asStableSnapshot` enables deterministic snapshots despite dynamic values
- `stopBrowser` helper ensures cleanup between test cases

**verdict**: clone directly as test infrastructure

## test patterns [REUSE]

### pattern 1: BDD structure with test-fns

```typescript
import { given, then, when } from 'test-fns';

describe('browser.start', () => {
  given('[case1] --mode HEADLESS', () => {
    const session = 'start-case1';
    const wsEndpointFile = getWsEndpointFile(session);

    beforeAll(() => stopBrowser(session));
    afterAll(() => stopBrowser(session));

    when('[t0] skill is invoked', () => {
      then('it starts browser and writes wsEndpoint file', () => {
        const result = rhxFull(`rhx browser.start --mode HEADLESS --session ${session}`);
        expect(asStableSnapshot(result.stdout)).toMatchSnapshot('stdout');
        expect(asStableSnapshot(result.stderr)).toMatchSnapshot('stderr');
        expect(result.stdout).toContain('🐢 surfs up');
      });
    });
  });
});
```

**why important**: consistent BDD structure, snapshot coverage, session isolation

### pattern 2: session isolation via unique session names

```typescript
given('[case1] success path', () => {
  const session = 'snapshot-case1';  // unique per case

  beforeAll(() => {
    stopBrowser(session);
    rhxFull(`rhx browser.start --mode HEADLESS --session ${session}`);
  });

  afterAll(() => stopBrowser(session));

  // tests run against isolated session
});
```

**why important**: parallel test execution, no cross-contamination

---

### pattern 3: error path test

```typescript
given('[case2] without --mode', () => {
  when('[t0] skill is invoked without --mode', () => {
    then('it fails with usage message', () => {
      let error: (Error & { stdout?: string; stderr?: string }) | null = null;
      try {
        rhxFull('rhx browser.start');
      } catch (e) {
        error = e as Error & { stdout?: string; stderr?: string };
      }
      expect(error).toBeTruthy();
      expect(asStableSnapshot(error?.stdout || '')).toMatchSnapshot('stdout');
      expect(asStableSnapshot(error?.stderr || '')).toMatchSnapshot('stderr');
      expect(error?.message).toContain('--mode required');
    });
  });
});
```

**why important**: captures both success and error path snapshots

### pattern 5: stealth verification

```typescript
when('stealth plugin is active', () => {
  then('navigator.webdriver is false', async () => {
    const page = await scene.session.context.newPage();
    const webdriver = await page.evaluate(() => navigator.webdriver);
    await page.close();
    expect(webdriver).toBe(false);
  });
});
```

**why important**: validates antibot effectiveness
